# Working preferences

## Writing style (applies to everything: chat replies, emails, docs, commit messages, code comments, UI copy)

- Never use em dashes or en dashes (— or –). Use a comma, a period, parentheses, or a colon instead. Rewrite the sentence if needed. This is a hard rule with no exceptions.
- Keep the tone human and natural, like a real person talking. Avoid stiff, corporate, or robotic phrasing. Write plainly and warmly.
